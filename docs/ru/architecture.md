# Архитектура

Английская версия: [Architecture](../architecture.md)

## Модель сервисов

Целевая система состоит из двух прикладных сервисов и одного edge-компонента:

- `telegram-bot`: собирает лиды из Telegram-диалогов
- `web-assistant`: собирает лиды с сайта и отдает защищенный UI просмотра заявок
- `caddy`: публичная HTTPS-точка входа для веб-трафика

## Границы системы

- Внешний трафик допускается только через `caddy`.
- `web-assistant` работает только по внутреннему HTTP за reverse proxy.
- `telegram-bot` может работать в режиме polling без входящих портов.

## Общая доменная модель

Оба канала используют общую модель `Lead`:

| Поле | Назначение |
|------|------------|
| `name` | Имя лида |
| `telegram_username` | Опциональный контакт |
| `phone` | Опциональный контакт |
| `email` | Опциональный веб-контакт |
| `request` | Текст запроса |
| `source` | `telegram_bot` или `website_assistant` |

Правило валидации: должен быть заполнен хотя бы один контакт. Для Telegram это `phone` или `telegram_username`, для сайта — `phone` или `email`.

Перед сохранением имя нормализуется (например, `"я Вовочка"` -> `"Вовочка"`). Если значение похоже на другой тип поля, пользователь остается на текущем шаге и получает повторный запрос.

### Контактная модель сайта

Website assistant собирает `phone` или `email` и сохраняет их как отдельные колонки в PostgreSQL. Для веб-канала `source` всегда равен `website_assistant`.

## Данные и наблюдаемость

- Основное хранилище: PostgreSQL с таблицами `leads` и `lead_events`.
- Чтение: `web-assistant` отдает `/api/leads` и `/leads` с токен-защитой.
- Диагностика качества данных: `lead_events.payload` хранит `qa_flags` и `offscript_count`.
- Логи: все сервисы пишут в stdout.
- Корреляция: по возможности используйте `lead_id` и/или `session_id` в логах.

## Поток диалога

- `telegram-bot`: команды `/start` или `/new` сбрасывают сессию и запускают сбор заявки.
- `web-assistant`: `POST /api/chat/start` создает `session_id` и запускает тот же четырехшаговый flow.
- На каждом шаге (`name`, `contact`, `request`, `confirm`) выполняется валидация до сохранения.
- Если введено значение не того типа, пользователь остается на текущем шаге и получает повторную подсказку + recovery-текст от AI.
- Финальное подтверждение сохраняет лид и сбрасывает состояние сессии.

## TLS-стратегия

TLS управляется напрямую через Caddy:

1. `CADDY_SITE_HOST` указывает на публичный домен VPS.
2. Caddy завершает HTTPS, делает редирект с HTTP на HTTPS и хранит ACME-данные в Docker volume.
3. `web-assistant` остается HTTP-only сервисом внутри Docker-сети.
